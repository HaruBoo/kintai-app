# Swell 勤怠管理アプリ

🇯🇵 日本語 | [🇺🇸 English](./README.en.md)

Excelベースの勤怠管理をブラウザ・デスクトップアプリで置き換えるための勤怠管理ツールです。

🔗 **[アプリを開く](https://kintai-app-swell.vercel.app)**

---

## 機能一覧

### 認証・セキュリティ

- **メールアドレス＋パスワードでログイン**（Supabase Auth）
- **ロールベースの画面切り替え**（管理者 / リーダー / 一般従業員）
- **タブ選択による画面制御**（一般タブ・管理者タブで表示が変わる）
- **不正ログインの防止**（一般ユーザーが管理者タブでログインしようとすると拒否）

### 一般従業員の機能

| 機能 | 説明 |
|---|---|
| 出勤・退勤打刻 | セレクトボックスで時間を選んで打刻 |
| 休憩時間の自動計算 | 実働6時間超で1時間を自動控除 |
| 休憩時間の手動変更 | セルをクリックして手動で修正・実働時間も自動再計算 |
| 備考欄の入力 | セルをクリックして備考を記入・保存 |
| 勤務区分の選択 | 現場 / 在宅 / 午前休 / 午後休 / 全休 / 有給 / 早退 から選択 |
| 月別データ管理 | ◀ ▶ ボタンで月を切り替え |
| 勤怠CSVダウンロード | Excel対応（BOM付きUTF-8） |
| 交通費申請 | 日付・出発駅・到着駅・金額を入力して一覧管理 |
| 領収書画像の添付 | 交通費に画像を添付、クリックで拡大表示 |
| ZIP一括ダウンロード | CSV＋領収書画像をまとめてZIPでダウンロード |
| 勤怠・交通費の一括提出 | 「提出」タブから当月分をまとめてリーダーへ提出 |
| 土日・祝日の色分け | 土曜＝水色、日曜＝ピンク、祝日＝オレンジ |
| ライト／ダーク／自動モード | 時間帯で自動切り替えも可能 |
| スマホ対応 | レスポンシブデザイン |

### リーダーの機能

| 機能 | 説明 |
|---|---|
| チーム勤怠の確認 | 社員を選択して月別の勤怠データを閲覧 |
| 承認・差し戻し | 提出済みデータを承認して管理者へ送付、または理由を添えて差し戻し |
| CSVダウンロード | 確認した社員・月のデータをCSV出力 |

### 管理者の機能

| 機能 | 説明 |
|---|---|
| ユーザー管理 | 社員の一覧表示・ロール変更（管理者 / リーダー / 一般） |

| 全社員の勤怠確認 | 社員を選択して月別の勤怠データを閲覧 |

## ログインについて

画面上部のタブで「一般」または「管理者」を選び、メールアドレスとパスワードを入力します。

> **注意：** パスワードの末尾にスペースが入っているとログインできません。入力後に不要なスペースが混入していないか確認してください。

| タブ選択 | アカウント種別 | 表示される画面 |
|---|---|---|
| 一般 | 一般ユーザー | 一般画面 |
| 一般 | リーダー | 一般画面＋「チーム承認」タブ |
| 一般 | 管理者 | 一般画面（管理者も一般画面を使える） |
| 管理者 | 管理者 | 管理者画面 |
| 管理者 | 一般ユーザー | ❌ ログイン拒否 |

## 使い方

### 出勤・退勤の打刻

1. 時間のセレクトボックスで打刻したい時間を選ぶ
2. **出勤** ボタンを押す
3. 退勤時に同様の手順で **退勤** ボタンを押す
4. 休憩時間と実働時間が自動で計算されます

> 打刻は当月のみ可能です。過去・未来の月は閲覧のみになります。

### 勤務区分の入力

- テーブルの**勤務区分**列のプルダウンから選択するだけで保存されます
- 選択肢：現場 / 在宅 / 午前休 / 午後休 / 全休 / 有給 / 早退

### 備考・休憩時間の編集

- テーブルの**備考欄**をクリック → テキストを入力 → Enter または「確定」で保存
- テーブルの**休憩時間**をクリック → `1:00` や `0:30` の形式で入力 → 確定すると実働時間も自動で再計算

### 交通費申請

1. 領収書画像を添付（任意）
2. 購入日・購入金額を入力（任意）
3. 日付・出発駅・到着駅・費用を入力
4. **追加する** ボタンで一覧に追加
5. 月末に **表をダウンロード**（CSV）または **ZIP（CSV＋画像）** でダウンロード

### 勤怠・交通費の提出

1. 「**提出**」タブを開く
2. 当月の勤務日数・交通費合計を確認する
3. **📤 提出する** ボタンを押す
4. 提出後は勤怠・交通費の編集ができなくなります
5. リーダーが確認・承認すると「管理者承認待ち」に変わります
6. 管理者が最終承認すると「✅ 最終承認済み」になります
7. 差し戻しされた場合は理由が表示されるので修正して再提出してください

### 3段階の承認フロー

```
社員が提出 → リーダーが確認・承認 → 管理者が最終承認
               （差し戻し可）            （差し戻し可）
```

### 管理者：勤怠確認

1. 管理者タブでログイン
2. **勤怠確認** タブを開く
3. プルダウンで社員を選択
4. ◀ ▶ で月を切り替えてデータを確認
5. **表をダウンロード** で該当社員・月のCSVを出力
6. **全社員まとめてダウンロード** で全員分を一括出力

## セキュリティ設計

- **RLS（行レベルセキュリティ）**：自分のデータのみ読み書き可能
- **Edge Functions**：管理者操作（招待・削除）はサーバー側で処理
- **ロール照合**：ログイン時にタブ選択と実際のロールを照合
- **PII保護**：社員番号・氏名はメモリのみ保持、DBには保存しない

## ローカルで動かす

```bash
# 依存パッケージのインストール
npm install

# .env ファイルを作成して Supabase の接続情報を設定
# VITE_SUPABASE_URL=...
# VITE_SUPABASE_ANON_KEY=...

# 開発サーバー起動（ブラウザ）
npm run dev

# デスクトップアプリとして起動（Tauri）
npx tauri dev
```
